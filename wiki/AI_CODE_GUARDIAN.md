# AI Code Guardian

This policy is mandatory before any AI or human-assisted coding in Agente RME AI or RME Workspace.

## Pre-edit proof

1. Read this wiki index and the pages relevant to the requested subsystem.
2. Inspect the current Canary/RME source or official OpenTibia material definition that owns the behavior.
3. Trace the active runtime entry point and consumer in the project being changed.
4. Name the evidence files before editing. If evidence is missing, do not implement a guess.

## Prohibited work

- Invented item IDs, flags, sprite properties, border orientation, brush behavior, OTBM nodes, or material families.
- Name-based classification when exact OTB, appearance, catalog, or material data exists.
- Placeholder success, silent fallbacks, mock engines in production paths, duplicate render/brush/map cores, and speculative compatibility layers.
- Copying geometry from reference maps. Only abstract, provenance-bearing style metrics may be learned.
- UI code that bypasses certified shared services or writes direct tile stacks.

## Required path

Semantic intent must flow through Planner, contextual material resolution, certified brush engines, atomic transactions, OTBM validation, visual QA, and human approval when applicable. Model output never has authority to write a concrete game ID directly.

## Completion proof

Run the real entry point and subsystem-specific checks. Do not claim parity or compatibility from syntax checks alone. Update this wiki and the Planner knowledge database only with verified facts and explicit provenance.
